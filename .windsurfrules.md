## GitNexus MCP - Code Intelligence & Knowledge Graph

### Cài đặt

```bash
cd tools/gitnexus/gitnexus
npm install
npm run build
```

### Cấu hình MCP

File `.mcp.json` tại thư mục gốc:
```json
{
  "mcpServers": {
    "gitnexus": {
      "command": "gitnexus",
      "args": ["serve"],
      "env": {
        "GITNEXUS_PORT": "4747",
        "GITNEXUS_REPO_PATH": "d:\\laragon\\www\\hethongquanlyktx"
      }
    }
  }
}
```

### Commands cơ bản

```bash
# Index codebase (chạy 1 lần đầu hoặc khi code thay đổi lớn)
npx gitnexus analyze

# Index lại với force (khi cần update ngay lập tức)
npx gitnexus analyze --force

# Serve MCP server (cho AI agents)
gitnexus serve

# Query knowledge graph
gitnexus query "tìm hiểu về auth logic"

# Context của một symbol
gitnexus context SinhvienController

# Impact analysis trước khi sửa code
gitnexus impact({target: "symbolName", direction: "upstream"})

# Detect changes trước khi commit
gitnexus detect_changes
```

### PowerShell Aliases

Thêm vào `$PROFILE`:
```powershell

Set-Alias gn-index gitnexus-index

Set-Alias gn-status gitnexus-status
```

### Khi nào dùng GitNexus?

- **Trước khi sửa code**: Chạy `gitnexus impact` để xem blast radius
- **Khi cần hiểu architecture**: Chạy `gitnexus query` để tìm execution flows
- **Khi debug bug**: Chạy `gitnexus context` để xem caller/callees
- **Trước khi commit**: Chạy `gitnexus detect_changes` để verify scope

## Code Quality Tools

### PHPStan - Static Analysis

**Cài đặt:**
```bash
composer require --dev phpstan/phpstan
```

**Cấu hình:** `phpstan.neon` (đã tạo)

**Commands:**
```bash
# Chạy static analysis
vendor/bin/phpstan analyse

# Chạy với memory limit cao hơn
vendor/bin/phpstan analyse --memory-limit=1G

# Chạy chỉ một file/folder
vendor/bin/phpstan analyse app/Services/StudentService.php
```

**Khi nào dùng:**
- Trước khi commit: Tìm type errors, null pointer risks
- Trước khi merge: Verify code quality
- Khi refactor: Kiểm tra không introduce bugs

**Level hiện tại:** 5 (trung bình, cân giữa strictness và practicality)

## Best Practices

### Trước khi sửa code

1. Chạy `gitnexus impact` để xem blast radius
2. Nếu HIGH/CRITICAL risk, cân nhắc lại
3. Nếu MEDIUM, review dependencies kỹ

### Sau khi sửa code

1. Chạy `gitnexus detect_changes` để verify scope
2. Chạy `vendor/bin/phpstan analyse` để tìm type errors
3. Chạy `vendor/bin/pint` để format code
4. Chạy tests (nếu có)
5. Chạy `/audit [area]` nếu thay đổi UI

### Trước khi commit

1. Review changes với `gitnexus detect_changes`
2. Chạy `vendor/bin/phpstan analyse` để verify code quality
3. Chạy `vendor/bin/pint` để format code
4. Chạy tests
5. Verify không có unintended side effects

 ## Karpathy LLM Coding Guidelines
 
 > Source: [forrestchang/andrej-karpathy-skills](https://github.com/forrestchang/andrej-karpathy-skills)
 
 ### 1. Think Before Coding
 - State assumptions explicitly. Ask when uncertain.
 - Present multiple interpretations instead of picking silently.
 - Stop when unclear — name what's confusing, then ask.
 
 ### 2. Simplicity First
 - Write the minimum code that solves the problem.
 - No speculative features, abstractions for single-use code, or unrequested "flexibility".
 - If 200 lines could be 50, rewrite it.
 
 ### 3. Surgical Changes
 - Touch only what the request requires.
 - Don't improve adjacent code, comments, or formatting.
 - Remove imports/vars YOUR changes made unused; leave pre-existing dead code alone.
 - Every changed line must trace to the user's request.
